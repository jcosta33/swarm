## TL;DR

Audits inventory **truth of the codebase now** mapped to explicit rules (`constitution`, ADRs, security canon). They're not redesign documents — prescribing belongs in specs.

## Why audits gate refactors indirectly

Mechanics: audit spawns refactor / performance typed tasks preserving numbered issues (`medium` distortion budget vs research). Auditor skill forces traceable linkage — each finding cites observable mechanism + rule violated.

Poor audits produce Janitor archaeology sessions — excavation skill minimises unstructured debt prose.

## Epistemic guardrails encoded

| Risk | Skill counterweight |
|------|---------------------|
| Sneaking redesign | Explicit ban — push to spec-writing |
| Severity inflation | Calibration tables / exemplars inside scaffold SKILL |
| Unverifiable fluff | Observation requires **file:line** or reproducible trace |

Handoff Auditor → Skeptic deepen-audit when adversarial amplification needed.
