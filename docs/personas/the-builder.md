# Persona (documentation): The Builder

> **Runtime profile:** all constraints, forbiddances, proofs, red flags → [`/scaffold/.agents/skills/personas/SKILL.md`](../../scaffold/.agents/skills/personas/SKILL.md) (**The Builder** section).

## Cognitive slot in the taxonomy

Executes additive / integrative coding once Architect truth is frozen. Holds **externalised contract discipline** (“spec is oracle”) to counter agentic completion bias (“just infer the nicest requirement”).

## Distinct from neighbours

| Neighbour | Why not interchangeable |
|-----------|--------------------------|
| **Architect** | Builder consumes truth tables; rewriting them mid-flight is rerouting disguised as progress. |
| **Janitor** | Structural neutral refactors forbid behaviour deltas—Builder thrives on observable capability change. |
| **Skeptic** | Builder optimises delivery trust; Skeptic optimises falsification loops—mixed incentives in one seat produce false merges. |

## Default routing footprint

Anchors `feature`, `rewrite`, `integration`, many `kickback` waves—see [`compatibility-matrix`](../reference/compatibility-matrix.md).

## Structural failure signatures

Symptomatic patches shipped as specs; silent helper reinvention fracturing idioms; “done” narration absent raw command receipts—signals profile enforcement drift or missing scaffold sync.

## Custom overlays (optional)

Heavy TypeScript ergonomics shops may splice **Type Surgeon** overlays; SDK plumbing heavy teams augment with **Integrator** framing—patterns described in [`customizing-personas`](../guides/customizing-personas.md), still expressed as additional SKILL material *you own*, not edits to canonical thirteen without ADR-equivalent rationale.
