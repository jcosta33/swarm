Two further notes the spec author must hold throughout:

## Theme 1 — SOL surface syntax & grammar (Q1.1–Q1.7)

**Q1.1 — Block delimiters.** Canonical: **bare-header form `BLOCK-TYPE ID:`** (e.g. `REQ AC-001:`); body = contiguous non-blank lines until the next block header, blank-line gap, or markdown heading. Fenced `:::TYPE … :::` is rejected as v0.1 normative; it MAY later be an optional editor-robustness alias only.
*Evidence:* EARS/FRETish/Gherkin all use leading-keyword bare lines; build brief uses bare headers throughout (§6.4–6.18); fenced `:::`+in-block YAML is malformed in source and fragile to parse.
*Sources change:* `sol.md` drops `:::REQ…:::END` and in-block YAML, converts to bare headers + inline keyword clauses, fixes malformed examples. `swarm-sol.md` replaces significant-indentation (Indent/Dedent) with the line-grouping rule. `compiler.md` adds the mandatory trailing colon.

**Q1.2 — Verification keyword.** Canonical: **`VERIFY BY`** (two words, uppercase, space-separated); `BY` is mandatory; `VERIFY_BY` is surface-illegal. `verify_by` is the IR field name only. (Composes with Q4.14 reference grammar per X3.)
*Evidence:* SOL is English-shaped CNL inside Markdown; every other multi-word keyword is space-separated (`MUST NOT`, `OWNED BY`, `DEPENDS ON`); RFC 8174 spells multi-word operators as separate uppercase words.
*Sources change:* `swarm-sol.md` renames surface `VERIFY_BY→VERIFY BY` (keep `verify_by` IR field). `compiler.md` adds mandatory `BY`. `sol.md` replaces `verify:` YAML field with inline clause.

**Q1.3 — Block-type set.** Canonical: **exactly seven** — `REQ, CONSTRAINT, INVARIANT, INTERFACE, QUESTION, TRACE, VERDICT`. Full words (not POLICY/INV). `TASK-MAP`, `FINDING`, `ADR` are **not** block types (they are downstream artifacts/passes). Obligation blocks = REQ/CONSTRAINT/INVARIANT only; INTERFACE declares boundaries; QUESTION marks ambiguity; TRACE claims; VERDICT judges.
*Evidence:* build brief §6.3/§3 fixes the seven; ASD-STE100 one-word-one-meaning kills abbreviations; INCOSE behavior-vs-restriction-vs-invariant distinction; pipeline doctrine keeps planning/memory downstream.
*Sources change:* `sol.md` removes TASK-MAP block. `swarm-sol.md` replaces `REQ|INV|POLICY|TASK|FINDING|ADR` with the seven; both add INTERFACE/QUESTION as first-class.

**Q1.4 — Modal set.** Canonical: **MUST, MUST NOT, SHOULD, SHOULD NOT, MAY** (uppercase only). `SHALL`/`SHALL NOT` **removed**. `CAN`/`WILL` are non-modal and forbidden in binding clauses (lint warning). SHOULD/SHOULD NOT require an accompanying `BECAUSE` or `EXCEPT`. Lowercase `must/should/may` carry no force (plain prose).
*Evidence:* Federal Plain Language + FRCP eliminated "shall" (courts read it as may/will/must); RFC 8174 makes MUST≡SHALL (so SHALL is redundant) and gives force only to all-caps; ASD-STE100 one-word-one-meaning.
*Sources change:* `sol.md` removes SHALL/SHALL NOT and rewrites every SHALL example→MUST (incl. IR `modality` fields). build brief removes SHALL from its §6.11 table, adds SHOULD NOT. `swarm-sol.md` adds SHOULD NOT + BECAUSE/EXCEPT gate, renames `MUST_NOT→MUST NOT`.

**Q1.5 — ID convention.** Canonical: **per-type short IDs** `TYPE PREFIX-NNN:` — REQ→`AC-NNN`, CONSTRAINT→`C-NNN`, INVARIANT→`I-NNN`, INTERFACE→`IF-NNN`, QUESTION→`Q-NNN`, TRACE→`T-NNN`; VERDICT reuses the judged obligation's ID. IDs unique within a spec; cross-spec ref qualifies as `spec-id:AC-001`. Dotted-semantic IDs and single spec-prefix `CO-NNN` rejected at the surface. IR node id MAY be namespaced `REQ.<spec>.AC-001`.
*Evidence:* build brief uses per-type prefixes everywhere; `SOL-S005` presupposes prefix-per-type; INCOSE/Femmer want stable opaque traceability keys decoupled from text; dotted form belongs to the IR namespace.
*Sources change:* `sol.md` dotted-semantic→type-prefixed surface (keep dotted only as IR id). `swarm-sol.md` `CO-NNN`→per-type prefixes; constrain to `<PREFIX>-<digits>`. `compiler.md` standardize on role prefixes (domain tags optional infix).

**Q1.6 — Clause keywords.** Canonical REQ order: `WHERE → WHILE → WHEN → IF [THEN] → THE <actor> <MODAL> <response> → [AND THE <actor> <MODAL> <response>]* → [BECAUSE] → [EXCEPT] → VERIFY BY` + trailing metadata `DEPENDS ON / TOUCHES / WRITES / READS / AFFECTS / RISK <low|medium|high|critical>`. `THEN` is optional sugar after `IF` only (EARS unwanted-behavior), never after WHEN/WHILE. INVARIANT: `<property> MUST|MUST NOT <hold>` (no ALWAYS/NEVER). INTERFACE: `RETURNS, ACCEPTS, ERRORS, OWNED BY`. QUESTION: `[blocking|non-blocking]` tag + `AFFECTS`. TRACE: `IMPLEMENTS, PRESERVES, CHANGED, PROOF`. VERDICT: `REASON, EVIDENCE`. Removed from v0.1: `ALWAYS/NEVER, EXPOSES/INPUT/OUTPUT, MAP/TO/ORDER/ASK`. Timing (`WITHIN/BEFORE/UNTIL/IMMEDIATELY/EVENTUALLY`) **deferred to v0.2** with FRETish temporal-logic semantics.
*Evidence:* build brief §6.12–6.16; EARS condition keywords (When/While/Where/If-Then); FRETish field-optionality + timing requires real temporal semantics to be sound; ASD-STE100 removes redundant ALWAYS/NEVER.
*Sources change:* `sol.md` remove ALWAYS/NEVER, EXPOSES/INPUT/OUTPUT, MAP/TO/ORDER/ASK; move timing to v0.2; reconcile SOL101/SOL102 to "THEN required only after IF". `swarm-sol.md` add WHERE + the missing metadata/interface/trace/verdict keywords.

**Q1.7 — One normative EBNF.** Canonical = the single line-oriented EBNF from Cluster 1 (bare-header + following-lines; uppercase case-sensitive keywords; lowercase = prose; conditions are opaque text in v0.1, expression sublanguage deferred to v0.2). **Two amendments to make it consistent with X1–X3:**
- `verify_line = "VERIFY BY" , ws , verify_ref , nl` where `verify_ref = proof_type , ":" , adapter , ":" , artifact , [ "#" , selector ]` and `proof_type ∈` the closed 9-set (Q4.14). *(Was opaque `verification_ref`; now typed per X3.)*
- `verdict_value = "PASS" | "FAIL" | "BLOCKED" | "UNVERIFIED"` (core), optionally followed by `"(" , lifecycle , " by " , authority , ": " , reason , ")"` where `lifecycle ∈ {WAIVED, STALE, CONTRADICTED}` (per X2/Q4.13).
- Lint constraints renumbered into the unified namespace (per X1): `SOL-S001` (precond with no actor_clause), `SOL-S003` (actor_clause with no modal), `SOL-S005` (prefix↔type mismatch), `SOL-P003`/`SOL-P004` (modal/overload at prose layer), `SOL-V001` (missing `VERIFY BY`, was SOL-S007), `SOL-V…`(bad VERDICT value, was SOL-S010). SHALL/SHALL NOT/CAN/WILL not valid modals.
*Sources change:* supersedes all three competing grammars (`sol.md` fenced+YAML, `swarm-sol.md` Indent/Dedent, `compiler.md` colon-less). Each adopts the seven types, per-type prefixes, no SHALL, `VERIFY BY`, no ALWAYS/NEVER/EXPOSES/MAP/timing, line-grouping, and the unified lint codes.

**Theme-1 residual gaps:** expression sublanguage (v0.2); timing semantics binding to proofs (v0.2, needs Theme 5); cross-spec ID import syntax; whether `AND THE` chaining violates singularity (lint decision: **permitted, lowered into multiple IR obligations, one warning if >2** — settled below in §11 gap G3); INTERFACE verification requirement (settled: **see Theme 5 gap-fill below — INTERFACE requires a `contract` VERIFY BY**).

## Theme 2 — APS controlled prose, requirement smells, unified lint taxonomy (Q5.16–Q5.17)

**Q5.16 — Unified lint namespace.** Canonical: **one prefix `SOL`, five layer letters, form `SOL-<LAYER><NNN>`**, each layer a 100-block, append-only with tombstoning:
1. `SOL-S###` SYNTAX (parser-detectable well-formedness)
2. `SOL-P###` PROSE (controlled-prose / requirement-smell, single-obligation-local) — the former APS layer; **`SOL-L`→`SOL-P` rename**
3. `SOL-M###` SEMANTIC (cross-reference: duplicate id, contradiction, unbound ref)
4. `SOL-V###` VERIFICATION (proof-binding: missing/stale/non-observable proof)
5. `SOL-O###` ORCHESTRATION (planning/parallelism: write-conflict-marked-parallel, dep cycle, blocking QUESTION reaching lowering)

`APS-` retired as a code prefix. Merge map: `SOL-S007/S010→SOL-V`; `SOL-S008→SOL-O`; `SOL-M003/M007/M008→SOL-V`; `SOL-M009→SOL-O301`; whole `APS-*` family→`SOL-P/M/V/O`; `sol.md` flat `SOL00x/10x→SOL-S`, `SOL20x→SOL-M/V/O`, `SOL30x→SOL-P`; `swarm-sol.md SOL-L→SOL-P`. Every diagnostic record carries `{code, severity, layer, span, message, suggest}`.
*Evidence:* one tool (`lint-spec`) → one greppable namespace; layers mirror compiler passes 1:1; ISO 29148 nine-characteristics partition; Femmer smell families = the P layer; ESLint/Clippy append-only-with-tombstone convention.

**Q5.17 — APS rule set with BLOCKING/ADVISORY severities.** Canonical: a rule is **BLOCKING** iff the defect changes *what* gets built (incomplete/non-binding/untestable/ambiguous); **ADVISORY** if it only affects *how it reads*.
- **BLOCKING:** `SOL-P001` dangling condition (trigger w/ no modal consequence), `SOL-P002` missing actor, `SOL-P003` missing/informal modality, `SOL-P004` bundled/overloaded obligation (was APS-O001), `SOL-P005` vague-quality word with no same-line observable criterion (was APS-A001), `SOL-P006` undefined term in binding clause, `SOL-P007` negation ambiguity (pair bare `MUST NOT` with affirmative), `SOL-P008` uncaptured behavioral uncertainty not lifted to QUESTION (was APS-Q001). Cross-layer blocking companions: `SOL-V001` no verification path (was APS-V001), `SOL-M001` actor/object incompleteness, `SOL-M002` contradiction (was APS-X001).
- **ADVISORY:** `SOL-P050` pronoun, `SOL-P051` passive voice, `SOL-P052` sentence length (>~20 words), `SOL-P053` non-present/non-active, `SOL-P054` prose noise (was APS-P001), `SOL-P055` redundancy (was APS-R001), `SOL-P056` comparative/superlative without baseline (**blocking if in a binding clause, advisory in commentary**).
- **HIGH-RISK WORD list** = union of brief's subjective/promotional list + Femmer loopholes & comparatives + Tjong/Berry quantifiers/connectives (full list adopted verbatim from Cluster 2).
- **"Same line makes it observable" rule** adopted verbatim: a high-risk word is permitted only when the same sentence/bullet/immediately-following line converts it to observable behavior (actor+action+object, measurable threshold, or named verification target); else `SOL-P005` blocking, fixed by a named `Concretize`/`Quantify` op — never open-ended rewrite.
- **`improve` is a closed operation set** wired to lint codes (the 10-op set is finalized under Q6.19 below; the Q5.17 op→code mapping uses those 10 names).
*Evidence:* compiler.md hard-error-vs-style-heuristic split; EARS mandates `<system response>`; 2025–26 negation-inversion research (grounds SOL-P007); ISO 29148 unambiguous/singular/verifiable; plain-language guides for the advisory style heuristics; ASD-STE100 for the glossary/undefined-term rule.
*Sources change:* build brief §8.8 re-labeled with severities and reissued under `SOL-P/M/V/O`; its high-risk list extended with loopholes/quantifiers/comparatives; its §8.7 improve table becomes canonical (wired to codes). `sol.md` SOL301/303→blocking, others advisory. `swarm-sol.md` hygiene table gains explicit blocking/advisory tags.

**Theme-2 residual gaps:** project-tunable severity (promote/demote via a `swarm.config` waiver schema — undefined, see §11 G1); the **binding-clause vs commentary boundary** (settled: a span is *binding* iff inside a typed obligation block, *commentary* otherwise — §11 G2); glossary/term-resolution precedence (in-file `TERM` vs `memory/glossary.md` — see Q8.26); a labeled gold corpus for SOL-P precision/recall (ship it, Q9.31).

## Theme 3 — LLM evidence: prose density, soft-control boundary, structured output (feeds Q3/Q4 + prose limits)

**Q-density-1.** Canonical: always-loaded normative artifacts (AGENTS.md/CLAUDE.md/kernel bootloader prose) **hard cap ≤200 lines / ≤25 KB**, soft target ~50–150 lines, gap-filling content only; everything procedural/conditional moves to lazily-loaded pass guides/profiles/refs. Frame the rationale as "**minimize always-on density to protect adherence and control cost**," **not** "models can't follow many instructions."
*Evidence:* Anthropic memory docs (official ≤200-line adherence boundary; auto-memory loads first 200 lines/25 KB); AGENTS.md efficiency study (−28.6% runtime); ETH bloat-vs-gap-filling study (bloat −3% success/+20% cost; gap-filling +4%/+14–22% tokens). **Currency note:** IFScale "68%@500" is superseded (2026 Arize re-run ≈99%@5,000, inflection ~2,000) — do **not** anchor the cap on a capability ceiling.
*Sources change:* `swarm-sol.md`/`sol.md` stop citing IFScale 68%@500 as load-bearing; re-anchor on the durable adherence/cost mechanism. build brief adds the **numeric cap** to its "keep AGENTS.md short" rule (currently an `SOL-P005`-class vagueness in its own terms).

**Q-semantics-1.** Canonical binding invariant: **all load-bearing meaning** (modality, actor, trigger/state, verification binding, authority order, conflict resolution, trace schema) lives in **SOL + the typed IR**, never in prose/skills/personas/AGENTS.md. Prose and skills are non-authoritative delivery layers.
*Evidence:* prompt-format sensitivity (±40% same content), multi-turn reliability decay (−39%), lost-in-the-middle / context-rot (20–50% degradation), and Anthropic's explicit "no guarantee of strict compliance" for CLAUDE.md.
*Sources change:* all three already assert this; each must (a) cite the format-sensitivity + multi-turn + context-rot triad as the *why* (not the softened density figure), and (b) add a regression check that **no skill, persona, or AGENTS.md section defines modality, authority order, or verification semantics.**

**Q-enforcement-1 (Q4 boundary).** Canonical hard boundary: Swarm prose/SOL/APS/skills/AGENTS.md are **SOFT control** (context/guidance) and MUST never be presented as enforcement. Anything that must hold regardless of model (write-surface gates, stop conditions, secret redaction, CONSTRAINT/INVARIANT enforcement, proof-required gates) is specified as **deterministic checks outside the model** (hooks/CI/permission/schema), where the environment supports them. **Structured/schema-valid output ≠ verification** (shape ≠ truth); every completion claim maps to independent deterministic/evidentiary verification; "tests passed" without output is an invalid proof. The spec states plainly that **today the enforcement lane is aspirational/manual** (markdown-only, no runtime) and must not claim enforcement exists.
*Evidence:* Anthropic ("context, not enforced configuration"; "use a PreToolUse hook"); OpenAI Structured Outputs ("doesn't prevent mistakes within values"); multi-turn unreliability corroborates external-gate need.
*Sources change:* build brief pairs its "do not claim enforcement" line with a **positive soft/hard boundary** and a named (currently-manual) enforcement lane mapping CONSTRAINT/INVARIANT/stop-rules/secret-redaction → hooks/CI/permission/schema. `swarm-sol.md`/`sol.md` CI/hook statements promoted verbatim into the kernel; the "schema-valid ≠ correct" point becomes a first-class rule in the verification contract (Theme 5).

**Theme-3 residual gaps:** dated "evidence as of 2026-05" caveat policy for any numeric capability claim; a first-class **enforcement-lane artifact** mapping each CONSTRAINT/INVARIANT/stop-rule to its eventual deterministic home (§11 G4); unify "no completion without evidence" + "schema-valid ≠ correct" into one verification-contract section.

## Theme 4 — Spec-driven framing + canonical PASS set (Q6.18–Q6.19)

**Q6.18 — Two-level model.** Canonical:
- **7 PHASES** (conceptual compiler stages, fixed/ordered): `PARSE → NORMALIZE → LOWER → EXECUTE → VERIFY → REVIEW → PROMOTE`.
- **9 PASSES** (schedulable transformations, in pipeline order): `author → lint → improve → lower → decompose → implement → verify → review → promote`. Pass→phase mapping: author=entry(pre-parse); lint=PARSE+NORMALIZE; improve=NORMALIZE; lower=LOWER; decompose=LOWER; implement=EXECUTE; verify=VERIFY; review=REVIEW; promote=PROMOTE.
- **5 STDLIB pass guides** (first to be tooled, a *subset* of the 9): lint, decompose, implement, review[profile: skeptic], promote.
*Evidence:* compiler theory (phase=stage, pass=traversal; "several phases can group into one pass"); Spec Kit's real order (constitution→specify→**clarify**→plan→tasks→**analyze**→implement) shows lint/improve and a pre-execute consistency gate as first-class; Kiro waves; SDD survey Specify→Plan→Implement→Validate.
*Sources change:* build brief adds the phase layer + pass→phase mapping. compiler.md table relabeled "phases," cross-referenced to the 9 passes. `swarm-sol.md` retitles its 5 to "STDLIB pass guides (subset)" and renames to canonical pass names (`adversarial-review→review[profile: skeptic]`, etc.).

**Q6.19 — Closed `improve` set = exactly 10 operations**, strictly semantics-preserving (intent change routes to amendment/review, never improve): `NORMALIZE, ATOMIZE, CONCRETIZE, QUANTIFY, BIND (merges old Bind+Trace), SCOPE, CLARIFY, DECONFLICT, COMPRESS (merges old Compress+Stabilize), PROMOTE`. **`decompose` is NOT an improve op** (it's a pass). Each op has explicit pre/post and maps to lint codes (renumbered per X1): `NORMALIZE→SOL-P003/SOL-V…`; `ATOMIZE→SOL-P004`; `CONCRETIZE→SOL-P005`; `QUANTIFY→SOL-P005`; `BIND→SOL-V001/SOL-V…`; `SCOPE→SOL-O…`; `CLARIFY→SOL-P008`; `DECONFLICT→SOL-M002`; `COMPRESS→SOL-P054/SOL-P055`; `PROMOTE→`promotion protocol.
*Evidence:* reconciles brief's 12 (drop redundant Bind/Trace, Compress/Stabilize pairs) and compiler.md's 8 (excise decompose, split concretize from clarify); Femmer/Paska/EARS-CTRL pattern = detect-specific-defect → structured-repair-with-exit-condition; Spec Kit /clarify + /analyze = deployed CLARIFY + DECONFLICT.
*Sources change:* brief §8.7 12→10 (merge Trace→Bind, Stabilize→Compress); compiler.md remove decompose, rename `bind-proof→bind`, add concretize/compress/promote.

## Theme 5 — Verification model: verdicts, proof taxonomy, drift (Q4.13–Q4.15, Q9.29)

**Q4.13 — Verdict set = 7 values, core+lifecycle (this is X2, normative everywhere).**
- **Core (4, mutually exclusive run results):** `PASS` (bound proof ran, succeeded), `FAIL` (ran, failed), `BLOCKED` (couldn't run — prereq/tool/env missing), `UNVERIFIED` (no acceptable proof bound / none executed).
- **Lifecycle (3, decorate a core outcome):** `WAIVED` (FAIL/UNVERIFIED accepted with named authority + reason + expiry), `STALE` (prior PASS whose evidence no longer matches current source hash / changed surfaces), `CONTRADICTED` (two proofs disagree, or trace/code disagrees with obligation).
- Verdict line: `VERDICT <id>: <CORE>` optionally `(<lifecycle> by <authority>: <reason>)`. **Merge gate:** merge iff every required obligation is `PASS` or `WAIVED`, none `STALE/CONTRADICTED/FAIL/BLOCKED/UNVERIFIED`. Lint (was SOL-S010) accepts the core 4 and enforces lifecycle mandatory fields (WAIVED→authority+reason+expiry; STALE→prior-verdict ref+changed-surface; CONTRADICTED→two conflicting evidence refs).
*Evidence:* swarm-sol verdict table; WAIVED needed by ADR 0023/0024 exception channel; STALE = the machine-readable form of Q9.29 drift (the only honest verdict under "no runtime"); CONTRADICTED grounded by LLM-judge bias research.
*Sources change:* `sol.md` replaces 4-value §6.18 grammar + SOL-S010; `swarm-sol.md` keeps 7 but adopts core-vs-lifecycle framing; docs `verification-gates.md`/`flow-graph.md` add the enum; ADR 0021 references it.

**Q4.14 — Closed proof taxonomy = 9 types** (reconciles sol.md's 11 + swarm-sol's 7): `static, test, contract, property, model, perf, security, manual, monitor`. `unit/integration/e2e` are **scope qualifiers** under `test` (`test:e2e:`, `test:integration:`), not separate types; `runtime→monitor`. Binding syntax (this is X3): `VERIFY BY <type>:<adapter>:<artifact>[#selector]` — type is the closed lint/IR-typed dimension; adapter/artifact are project free-strings.
*Evidence:* closed types let lint enforce "INVARIANT prefers property/model/static" and let the merge gate weight by proof strength; VERINA (4.9% proof success ceiling) → `model` means model-checking OR economical proof, not theorem-per-obligation; Gherkin observable-outcome discipline folds into a `test`/`contract` lint; `manual` is the honest escape hatch (NL→formal undecidable).
*Sources change:* `sol.md` collapse 11→9 (demote unit/integration/e2e to scopes, runtime→monitor). `swarm-sol.md` extend 7→9 (add perf, security, contract/monitor granularity). `verification-gates.md` re-express named gates as AGENTS.md > Commands bindings of the 9 types.

**Q4.15 — Two-layer unification: SOL `VERIFY BY` = obligation-level binding; AGENTS.md > Commands = project-level adapter it resolves through.** (1) proof type+adapter resolves to a `{{cmd*}}` slot (`static→cmdLint/cmdTypecheck/cmdValidate`, `test→cmdTest`, `perf→cmdBenchmark`, `contract`/`security→integration-boundary cmds`). (2) The per-task-type matrix in `flow-graph.md` is re-framed as the default `(proof-type, phase)` suite per task type, binding to cmd* slots. (3) ADR 0022 spec-intent gates **are** VERIFY BY bindings (acceptance criterion→`VERIFY BY test:…`; behaviour-preservation→`VERIFY BY property|contract:…`). (4) Each proof→one VERDICT; the hard gate becomes "one VERDICT per required VERIFY BY binding"; ADR 0023 re-runs the resolved cmd*; ADR 0026 checks binding+verdict completeness. **ADRs 0021–0026 are subsumed (not replaced):** 0021=typed suite, 0022=intent bindings, 0023=enforcement, 0024=confidence tier of the verdict, 0026=conformance.

## Theme 6 — Multi-agent orchestration & safe parallelism (Q8.27, ties to Q2)

## (a) Canonical vocabulary cheat-sheet

**Block types (7):** `REQ` (id `AC-NNN`) · `CONSTRAINT` (`C-NNN`) · `INVARIANT` (`I-NNN`) · `INTERFACE` (`IF-NNN`) · `QUESTION` (`Q-NNN`) · `TRACE` (`T-NNN`) · `VERDICT` (reuses judged obligation id). Header form: `TYPE PREFIX-NNN:`. Obligation blocks (carry force) = REQ/CONSTRAINT/INVARIANT only.

**Modals (5):** `MUST`, `MUST NOT`, `SHOULD`, `SHOULD NOT`, `MAY` (uppercase only; SHOULD/SHOULD NOT require BECAUSE/EXCEPT). Removed: `SHALL`/`SHALL NOT`. Non-modal/forbidden in binding clauses: `CAN`, `WILL`.

**Clause keywords (uppercase, case-sensitive):** `WHERE WHILE WHEN IF [THEN] THE … AND THE … BECAUSE EXCEPT VERIFY BY DEPENDS ON TOUCHES WRITES READS AFFECTS RISK` (REQ); `RETURNS ACCEPTS ERRORS OWNED BY` (INTERFACE); `[blocking|non-blocking] AFFECTS` (QUESTION); `IMPLEMENTS PRESERVES CHANGED PROOF` (TRACE); `REASON EVIDENCE` (VERDICT). Deferred to v0.2: `WITHIN BEFORE UNTIL IMMEDIATELY EVENTUALLY` (timing).

**Verdicts (7 = 4 core + 3 lifecycle):** core `PASS` · `FAIL` · `BLOCKED` · `UNVERIFIED`; lifecycle decorators `WAIVED` (auth+reason+expiry) · `STALE` (source/surface hash drift) · `CONTRADICTED` (proofs disagree). Line: `VERDICT <id>: <CORE> [(<lifecycle> by <authority>: <reason>)]`. Merge gate: all required obligations `PASS` or `WAIVED`.

**Proof types (9, closed):** `static · test · contract · property · model · perf · security · manual · monitor`. Binding: `VERIFY BY <type>:<adapter>:<artifact>[#selector]`; `<adapter>` resolves through AGENTS.md > Commands (cmd* slots). Test scope qualifiers: `test:unit:`, `test:integration:`, `test:e2e:`. Proof-strength order (for CONTRADICTED tie-break): `model > property/contract > test > static > manual/monitor`.

**Phases (7, conceptual):** `PARSE → NORMALIZE → LOWER → EXECUTE → VERIFY → REVIEW → PROMOTE`.
**Passes (9, schedulable):** `author → lint → improve → lower → decompose → implement → verify → review → promote`. **Stdlib pass guides (5):** lint, decompose, implement, review[profile: skeptic], promote.
**Improve ops (10, closed, semantics-preserving):** `NORMALIZE · ATOMIZE · CONCRETIZE · QUANTIFY · BIND · SCOPE · CLARIFY · DECONFLICT · COMPRESS · PROMOTE`.

**Artifact filenames:** compiler-visible (`.swarm.` infix): `*.swarm.md` (source) · `*.swarm.ir.json` · `*.swarm.plan.json` · `*.swarm.trace.md`. Plain `.md` working artifacts: `task.md · review.md · finding.md · adr.md · audit.md · research.md · bug-report.md · memory/INDEX.md · memory/glossary.md · memory/patterns/*.md`. **No `verdict.md`** (VERDICT is a block; `review.md` is its container). Kernel-required core templates (7): `spec.swarm.md, task.md, trace.md, review.md, finding.md, adr.md, memory/INDEX.md`.

**Lint-code namespace (one prefix, five layers):** `SOL-<LAYER><NNN>` — `S`=syntax · `P`=prose (was APS/`SOL-L`) · `M`=semantic · `V`=verification · `O`=orchestration. Append-only, tombstoned. `APS-` retired as a code prefix (APS = the prose-standard name only). IR `diagnostics[].code` and all fixtures cite this single scheme.

**IR envelope:** `{ meta, nodes[], edges[], diagnostics[], provenance }`; edge types `{depends_on, blocks, conflicts_with, verified_by, affects, implements, preserves}`; three version fields `meta.language` (discriminator, e.g. `SOL/0.1`) / `meta.version` (spec content) / `provenance.compiler_version`.

**Surface vs IR field casing:** surface = space-separated uppercase (`VERIFY BY`, `DEPENDS ON`, `OWNED BY`, `WRITES`, `READS`, `AFFECTS`); IR = snake_case (`verify_by`, `depends_on`, `writes`, `reads`, `affects`). Lock groups = named `SURFACE`s (no `locks` primitive). Surface ids = `AC-001`; IR node ids may be `REQ.<spec>.AC-001`.

## (b) Residual gaps requiring an author's judgment call

- **G1 — Severity-override/waiver config schema.** Default BLOCKING/ADVISORY is fixed (Q5.17), but no canonical `swarm.config` schema lets a project promote advisories→errors (strict mode) or demote a blocker with a recorded waiver. compiler.md anticipates "stricter linting." *Judgment needed: schema + waiver-recording format.*
- **G2 — Binding-clause vs commentary boundary (recommended resolution embedded).** Many `SOL-P` codes (comparatives, loopholes, most high-risk words) are blocking only inside a binding obligation. **Recommend: a span is "binding" iff inside a typed obligation block (REQ/CONSTRAINT/INVARIANT), "commentary" otherwise.** Confirm and pin in the spec.
- **G3 — `AND THE` chaining vs singularity (recommended resolution embedded).** The brief allows `[AND THE …]*`; swarm-sol's SOL-S003 forbids >1 modal clause. **Recommend: permitted, lowered into multiple IR obligations; emit a `SOL-P004`-adjacent *warning* when >2 chained obligations** (atomize suggested), not a hard error. Confirm.
- **G4 — Enforcement-lane artifact.** A first-class (currently-manual) artifact mapping each CONSTRAINT/INVARIANT/stop-rule/secret-redaction → its eventual deterministic home (PreToolUse hook / CI / permission deny / schema validator). Asserted but not operationalized in any source.
- **G5 — WAIVER lifecycle.** Who holds waiver authority (human only? Lead Engineer profile? spec owner?), and whether WAIVED auto-expires on the next source-hash change. **Recommend: human-or-spec-owner authority, auto-expire on source-hash change (prevents zombie waivers).** Confirm.
- **G6 — CONTRADICTED resolution protocol.** Tie-break beyond the proof-strength order: re-run both? escalate to manual? The cheat-sheet fixes the ordering; the *action* on CONTRADICTED at the merge gate needs pinning (likely: block + route to review with the stronger oracle authoritative).
- **G7 — READS conflict rule + SURFACE attributes.** read/read = parallel-safe; read/write on the same surface = conflict edge (conflict-serializability). Plus a `SURFACE x = … [append-only|integration|shared]` attribute so shared/global/append-only surfaces (lockfiles, CI config, manifests) aren't treated as ordinary write conflicts and don't trigger blanket staleness. **New lint code `SOL-O305`** ("owned path outside declared write surface") needed for the two-tier lowering check.
- **G8 — `*.swarm.plan.json` full schema.** Named-as-contract (Q3.11) but field set unreconciled (sol.md `plan_id/max_parallel/tasks[]{lane,writes,locks,depends,merge_safe}` vs swarm-sol `PlanTask{pass,profile,inputs,outputs,batch}`). Resolve with the Q3.12 method: graph envelope + rich task payload; drop `locks` (→surface).
- **G9 — "Universal workflow rule" promotion target.** §10 routes such promotions to AGENTS.md, but ADR 0017 + the ≤200-line bootloader cap say only persistent *facts* go to AGENTS.md and procedures go to pass guides. *Judgment needed: a tie-break defining what a "workflow rule" promotion actually becomes (likely a pass-guide edit + an AGENTS.md one-line pointer, not inline procedure).*
- **G10 — Frontmatter field-name normalization.** Pick one vocabulary/value format across all docs. **Recommend `swarm_language: SOL/0.1` (discriminator) + `aps_version: 0.1` + `spec_version: 0.1.0` (content).** This directly unblocks the Q9.30 three-field IR mapping.
- **G11 — Trace-provenance schema.** Pin the exact fields the drift join + conformance checker depend on: `{source_hash, per_surface_hash[], adapter, verdict, tier, origin_obligations[], origin_traces[]}` — settle alongside the verdict enum so Q9.29/Q8.26/Q4.13 all reference one schema.
- **G12 — Gold corpus precision/recall baseline.** SOL-P rules need a labeled good/bad prose fixture set (sol.md targets 0.90 precision / 0.85 recall); ship it with the Q9.31 corpus so the high-risk-word list's false-positive rate is measurable.
